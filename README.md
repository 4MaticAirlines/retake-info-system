# Retake Info System

> Веб-сервис мониторинга ликвидации академических задолженностей с
> визуализацией риска отчисления обучающихся.

Сервис принимает на вход PDF-выписку студента или название учебной группы и
возвращает список доступных пересдач (основная сессия, первичная пересдача,
комиссия) вместе с категорией риска отчисления по каждой задолженности
(низкий / средний / высокий).

В основе оценки риска — модель Random Forest, предсказывающая вероятность
провала каждой попытки сдачи, и формула произведения по оставшимся попыткам
до комиссии. Модель обучается на синтетическом датасете и переобучается
автоматически при загрузке кафедрой обновлённой статистики дисциплин.

## Возможности

- **Два режима поиска**: по учебной группе или по PDF-выписке студента.
- **Распознавание задолженностей**: автоматическое извлечение названий
  дисциплин и статусов («неявка», «неудовлетворительно», «не зачтено»)
  из PDF-выписки.
- **Сопоставление дисциплин**: устойчивое сопоставление названий из
  выписки и записей расписания на основе коэффициента Жаккара
  (порог сходства 0,45).
- **Фильтрация по группе**: автоматический отбор только тех записей,
  что назначены для учебной группы студента.
- **Категория риска**: классификация каждой задолженности по
  вероятности отчисления — низкий, средний, высокий (с цветовой
  маркировкой в интерфейсе).
- **Интерфейс кафедры**: загрузка обновлённой статистики дисциплин
  и автоматическое переобучение модели.

## Требования

- Python 3.10 или новее
- pip
- Минимально 200 МБ свободного места на диске
- Свободный TCP-порт (по умолчанию 8000)

Опционально (для публикации в интернет):

- Доменное имя
- Веб-сервер (nginx или аналог) для проксирования
- TLS-сертификат (Let's Encrypt)

## Установка

### 1. Клонирование репозитория

```bash
git clone <ваш-репозиторий> retake_info_system
cd retake_info_system
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
source venv/bin/activate          # Linux / macOS
# или
venv\Scripts\activate              # Windows
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Создание файла окружения

Скопируйте `.env.example` в `.env`:

```bash
cp .env .env
```

При желании отредактируйте `.env` под свои нужды (см. раздел
[«Конфигурация»](#конфигурация-env)).

## Запуск

### Локальный запуск для разработки

```bash
uvicorn app.main:app --reload
```

Сервер запустится по адресу <http://127.0.0.1:8000>. Браузер
автоматически перенаправит на страницу `/files/`.

### Запуск с доступом из локальной сети

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Сервис будет доступен по `http://<IP-вашего-ПК>:8000` со всех
устройств в той же сети.

### Запуск в production-режиме

Параметры по умолчанию читаются из `.env`:

```bash
uvicorn app.main:app --host $SERVER_HOST --port $SERVER_PORT --workers 4
```

## Конфигурация (.env)

Файл `.env` в корне проекта содержит параметры, которые можно менять
без правки кода.

### Параметры базы данных

```ini
DATABASE_URL=sqlite:///./retake.db
```

URL подключения к СУБД через SQLAlchemy. По умолчанию — локальный
SQLite. Для PostgreSQL пример:
`postgresql+psycopg2://user:pass@localhost:5432/retake_db`.

### Параметры сетевого развёртывания

```ini
SERVER_HOST=0.0.0.0
SERVER_PORT=8000
```

`SERVER_HOST=0.0.0.0` — сервер слушает все сетевые интерфейсы.
`SERVER_HOST=127.0.0.1` — только локальные подключения (только разработка).

### URL внешних ресурсов

```ini
UNIVERSITY_SESSION_URL=https://misis.ru/students/session/
UNIVERSITY_RETAKES_URL=
```

URL страниц вуза, с которых автоматически собираются Excel-файлы
расписания. Если пусто — автозагрузка отключена.

### Параметры домена

```ini
PUBLIC_DOMAIN=
ALLOWED_HOSTS=*
```

`PUBLIC_DOMAIN` — публичный домен сервиса. Заполните после привязки
домена. Пример: `retake.example.com`.

`ALLOWED_HOSTS` — список разрешённых хостов через запятую.
`*` — любые хосты (только для разработки). В production:
`localhost,127.0.0.1,retake.example.com`.

## Настройка домена и публикация в интернет

Полный сценарий выноса сервиса наружу: ПК-сервер → роутер →
домен → пользователи.

### Шаг 1. Подготовьте ПК как сервер

1. Установите проект (см. [«Установка»](#установка)).
2. В файле `.env` выставьте:

   ```ini
   SERVER_HOST=0.0.0.0
   SERVER_PORT=8000
   ALLOWED_HOSTS=localhost,127.0.0.1,<ваш-домен>
   PUBLIC_DOMAIN=<ваш-домен>
   ```

3. Откройте порт 8000 в брандмауэре ОС:
   - **Windows**: «Брандмауэр Windows» → «Дополнительные параметры»
     → правило для входящих подключений → новое правило для TCP-порта 8000.
   - **Linux**: `sudo ufw allow 8000/tcp`.

### Шаг 2. Прокиньте порт на роутере

В настройках роутера домашнего интернета добавьте правило проброса:

| Параметр | Значение |
|---|---|
| Внешний порт | 80 (HTTP) или 8000 |
| Внутренний IP | IP вашего ПК в локальной сети (например, 192.168.1.10) |
| Внутренний порт | 8000 |
| Протокол | TCP |

Узнайте IP своего ПК командой `ipconfig` (Windows) или `ip addr` (Linux).

### Шаг 3. Привяжите домен

1. Купите домен у регистратора (reg.ru, namecheap, godaddy и т. д.).
2. Узнайте свой публичный IP (поиск «мой IP»).
3. В личном кабинете регистратора добавьте A-запись:

   ```
   Тип:    A
   Имя:    retake (или @ для корневого домена)
   Значение: <ваш-публичный-IP>
   TTL:    3600
   ```

4. Подождите 15–60 минут, пока DNS обновится. Проверьте:

   ```bash
   nslookup retake.example.com
   ```

5. Внесите домен в `.env`:

   ```ini
   PUBLIC_DOMAIN=retake.example.com
   ALLOWED_HOSTS=localhost,127.0.0.1,retake.example.com
   ```

6. Перезапустите сервис.

### Шаг 4 (опционально). Настройте HTTPS через nginx

Сырой uvicorn не поддерживает HTTPS. Рекомендуется поставить
**nginx** как reverse proxy:

```nginx
server {
    listen 80;
    server_name retake.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name retake.example.com;

    ssl_certificate     /etc/letsencrypt/live/retake.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/retake.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Получение бесплатного TLS-сертификата:

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d retake.example.com
```

Если динамический IP — настройте DDNS (DynDNS, no-ip.com) или
используйте Cloudflare Tunnel.

## Использование

### Для студента

1. Откройте `https://<домен>/files/`.
2. Введите номер учебной группы (например, «БИВТ-22-СП-1») **или**
   загрузите PDF-выписку.
3. Получите список доступных пересдач, разделённый по этапам:
   основная / первичная / комиссия.
4. Для PDF-режима — посмотрите блок «Классификация риска отчисления»
   с цветной маркировкой.

### Для сотрудника кафедры

1. Откройте `https://<домен>/files/manage`.
2. Загрузите Excel-файл с расписанием пересдач — он автоматически
   попадёт в общую базу.
3. Для обновления модели риска загрузите Excel со статистикой
   дисциплин (по формату `data/disciplines_template.xlsx`) —
   модель переобучится автоматически.

## Переобучение модели

Модель переобучается автоматически при загрузке нового файла со
статистикой через интерфейс. Если нужно сделать это вручную:

```bash
# 1. Отредактируйте data/disciplines_template.xlsx
# 2. Пересоздайте обучающую выборку:
python scripts/generate_history.py

# 3. Оцените качество и сохраните графики/метрики:
python scripts/evaluate_model.py

# 4. Перезапустите сервер
```

Метрики сохраняются в `data/output/metrics.txt`. Графики:

- `data/output/roc_curve.png`
- `data/output/confusion_matrix.png`
- `data/output/feature_importances.png`
